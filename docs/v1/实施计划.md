# V1 分阶段实施计划

## 阶段划分

1. Phase 1（已开始并落地）  
   核心领域模型与规则引擎基础能力：
   - 地图与地块规则常量
   - 初始状态/初始背包
   - 建筑库存与物品效果
   - 动作额外消耗
   - 状态钳制与资源死亡判定
   - 夜晚 `X` 区存活率函数
   - 同轮动作排序规则（提交时间 -> 入房顺序）

2. Phase 2（待实现）  
   房间与对局应用层：
   - 6 人补齐 AI
   - 回合收集与锁定
   - 阶段推进与全员休息特殊推进
   - 终局判定（`ALL_DEAD / HUMAN_ALL_DEAD`）

3. Phase 3（待实现）  
   接口与通知机制：
   - FastAPI HTTP + WebSocket
   - 玩家视图投影（BR-037）
   - 事件流与历史窗口（最近 100 条）

4. Phase 4（待实现）  
   AI 策略层：
   - `choose_action(obs, action_mask)` 统一接口
   - `RuleBot` 与 `LLMPolicy`
   - 调用失败回退链路与可观测性
