# 技能设计与覆盖评估（V1）

## 1. 目标

本游戏核心目标是“活下去”。技能体系应优先服务生存决策，再考虑资源与冲突收益。

## 2. V1 已有技能覆盖

- `day_explore_collect`：白天探索与补给。
- `night_survival_control`：夜晚风险控制与保命。
- `loot_window_decision`：战利品窗口 `GET/TOSS` 决策。
- `emergency_survival`：濒死状态保命。

结论：
- 这 4 个技能可以支撑 V1 可运行与联调。
- 对“基础生存闭环”是够用的。

## 3. 当前缺口（建议）

- 非窗口冲突决策缺口：
  - 当前仅覆盖战利品窗口，未单独覆盖“是否主动攻击”的策略技能。
- 逃生路径决策缺口：
  - 缺少专门“高风险地形规避/脱离危险区域”的移动策略技能。
- 资源枯竭中局缺口：
  - 缺少“中期资源调度（先用/先拿/先探索）”的独立技能。

## 4. 建议新增技能（V1.1 可选）

- `combat_engage_decision`
  - 作用：非窗口场景下，判断是否发起 `ATTACK`。
  - 风格：仅在状态优势与收益明确时进攻，否则回避。

- `safe_path_escape`
  - 作用：在危险局面中优先做“安全移动/撤离”。
  - 风格：优先低风险格与低暴露路径。

- `resource_recover_plan`
  - 作用：资源偏低但未濒死时的恢复规划。
  - 风格：`USE` / `EXPLORE` / `TAKE` 的顺序化决策。

## 5. 上线建议

- V1：先用现有 4 技能上线，确保稳定。
- V1.1：按观测数据逐步引入新增技能，不一次性全开。
- 任一新技能必须满足：
  - `action_mask` 硬约束
  - `choose_action` schema 校验
  - 失败回退 `RuleBot`
