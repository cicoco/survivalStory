# V1 技术选型清单

## 1. 语言与运行时

- Python 3.11+
  - 理由：生态成熟、与 FastAPI/NiceGUI 兼容、便于快速迭代规则引擎与 AI 调用链路。

## 2. 项目与依赖管理

- uv
  - 用途：环境创建、依赖安装、锁定与命令执行（`uv sync` / `uv run`）。
  - 约束：统一使用 `pyproject.toml` 管理依赖与工具配置。

## 3. 服务端框架（Phase 3 开始）

- FastAPI
  - 用途：HTTP + WebSocket 接入层。
  - 理由：异步能力成熟，适合事件驱动与多人对局状态同步。

## 4. 客户端（Phase 3 开始）

- NiceGUI
  - 用途：Python 一体化 Web UI。
  - 理由：减少前后端分离复杂度，快速实现玩家面板与操作流。

## 5. 领域与数据建模

- 标准库 `dataclasses`
  - 用途：领域实体（`PlayerState`, `ActionEnvelope`）。
  - 理由：轻量、可读性高，适合规则引擎核心对象。

- Pydantic v2（规划）
  - 用途：API 请求/响应模型、动作入参校验、LLM 输出结构校验。
  - 状态：Phase 3/4 引入。

## 6. 测试与质量

- unittest（当前已落地）
  - 用途：Phase 1 规则单测。

- pytest（已纳入 dev 依赖）
  - 用途：后续扩展参数化测试、集成测试。

- ruff（已纳入 dev 依赖）
  - 用途：静态检查与风格统一。

## 7. AI 策略层（Phase 4）

- RuleBot
  - 用途：稳定兜底策略。

- LLMPolicy（OpenAI API）
  - 用途：模板路由 + 结构化动作生成。
  - 约束：失败必须回退 RuleBot。

## 8. 存储与部署

- V1：内存态
  - 用途：房间状态、动作队列、历史窗口。

- V2+：PostgreSQL（规划）
  - 用途：持久化战绩、回放索引、审计日志。
