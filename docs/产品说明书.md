末日废墟生存战（AI对抗版）产品说明书

一、游戏核心定位

6人回合制末日生存策略游戏，真人玩家与AI Agent同局对抗，所有角色唯一目标为存活；通过移动、探索、资源争夺、休息等策略决策，规避死亡风险。单局结束支持房间级可配置的两种模式：`ALL_DEAD`（全员死亡，含AI）与 `HUMAN_ALL_DEAD`（全部真人玩家死亡）。

二、房间匹配核心逻辑

1. 单房间固定人数：6人

2. 补位规则

- 玩家创建/加入房间后，系统实时统计真人玩家数量

- 若真人玩家＜6人，系统自动生成AI Agent补满至6人（例：1名真人+5名AI Agent）

- AI Agent为系统自主决策角色，无玩家操控，严格遵循生存逻辑行动

3. 身份构成：真人玩家（手动操作）+ AI Agent（系统自动决策）

4. 房间状态与离开规则

- 房间状态：
  - 等待中：房主尚未 `start`
  - 游戏中：房主已 `start`
- 进入规则：
  - 仅“等待中”房间允许 `join`
  - “游戏中”房间禁止新玩家加入
- `leave` 规则：
  - 房主在“等待中”离开：房间解散
  - 房主在“游戏中”离开：房间关闭、所有在线玩家退出、该局立即结束
  - 普通玩家在“等待中”离开：退出房间，可继续加入其他房间
  - 普通玩家在“游戏中”离开：退出当前对局，按死亡处理

5. 开房终局模式配置
- 房主创建房间时必须指定终局模式（`end_mode`）：
  - `ALL_DEAD`：房间内所有角色（真人+AI）全部死亡时结束。
  - `HUMAN_ALL_DEAD`：房间内所有真人玩家全部死亡时结束（AI是否存活不影响结束）。
- 默认推荐：`HUMAN_ALL_DEAD`。

三、名词解释

以下术语为全篇统一口径，后文若出现“回合/轮/阶段”等表述，以本节定义为准：

- 角色（Character）：本局内可行动单位的统称，包含真人玩家角色与 AI Agent 角色。

- 玩家（Player）：默认指真人玩家（手动操作）；如需包含 AI，会明确写“所有角色（真人+AI）”。

- 房主（Host）：房间创建者，拥有 `start` 与房间级配置权限。

- 角色信息快照（Character Snapshot）：玩家通过“探索”获得的“当前建筑角色列表”在该时点的记录；属于个人认知数据，默认长期保留，但可能失真/不实时。

- 物资快照（Resource Snapshot）：玩家通过“探索”获得的“当前建筑剩余物资”在该时点的记录；属于个人认知数据，默认长期保留，但可能失真/不实时。

- 探索者（Observer）：执行“探索”并获得快照的一方。

- 被观察方（Observed）：被探索者在快照中识别到的一方，不会自动收到“已被发现”提示。

- 对局：一场完整游戏，从房主 `start` 到触发终局结算结束。

- 一天（Day Cycle）：由两个阶段组成，顺序固定为“白天阶段 → 夜晚阶段”；当夜晚阶段结束后，该天结束，系统推进到“第二天白天阶段”（随后再进入第二天夜晚），按此循环。

- 阶段（Phase）：白天或夜晚中的一个。阶段内会发生多次轮次操作。

- 轮次（Round）：阶段内的一次操作结算单元。每个存活玩家在该轮次最多提交 1 个动作；全员提交（或超时补动作）后统一结算，进入下一轮次或推进阶段。

- 动作（Action）：玩家在轮次内提交的单次指令，例如 `MOVE / EXPLORE / USE / TAKE / REST / ATTACK`；攻击结算阶段还包含后续动作 `GET / TOSS`。

- 本玩家阶段结束（Phase Ended for Player）：当玩家“死亡”或执行“休息（REST）”后，该玩家本阶段结束；本阶段结束后不再参与该阶段后续轮次。

- 阶段推进（Phase Advance）：当“所有存活玩家”均已阶段结束（死亡或休息）时，立即推进到下一阶段（白天/夜晚切换）。

- 阶段固定消耗（Base Upkeep）：每名玩家在每个阶段仅扣除一次基础消耗（如水分/饱食固定值），不因阶段内轮次数增加而重复扣除。

- 终局模式（End Mode）：房间级结束判定配置，取值为 `ALL_DEAD` 或 `HUMAN_ALL_DEAD`。

四、核心产品逻辑

1. 基础规则

1. 地图规则

地图为9×9坐标网格（x=1~9，y=1~9），坐标格式为(x,y)，网格分布固定，具体矩阵如下：

[
  ["Q","Q","Q","Q","Q","Q","Q","Q","Q"],
  ["Q","X","W","X","X","J","X","J","Q"],
  ["Q","J","X","B","B","X","B","X","Q"],
  ["Q","X","S","M","X","W","X","J","Q"],
  ["Q","W","X","J","X","X","W","X","Q"],
  ["Q","B","X","X","X","W","J","J","Q"],
  ["Q","X","J","X","X","M","S","W","Q"],
  ["Q","X","X","X","J","B","X","B","Q"],
  ["Q","Q","Q","Q","Q","Q","Q","Q","Q"]
]

各区域含义及昼夜通用规则（核心保命重点）：

- Q=街道/围墙：昼夜进入均立即死亡，任何时段禁止进入

- X=空地：白天进入立即死亡；夜晚进入不立即死亡，但会大幅降低夜晚存活率

- 安全区（昼夜均无即时死亡，可正常活动）：J=居民楼、B=办公楼、S=超市、W=水源、M=药店

2. 物品规则

物品仅可通过探索、拿取安全区建筑获得，存入背包后可随时使用，无使用次数限制（消耗后背包对应物品减少），各类物品效果固定（昼夜通用）：

- 面包：使用后饱食+10

- 瓶装水：使用后水分+10

- 压缩饼干：使用后饱食+20

- 罐头：使用后饱食+20

- 桶装水：使用后水分+20

- 清水：使用后水分+15

补充说明：物品无过期、损坏机制，拿取后永久存于背包，使用后立即生效，无额外消耗。

3. 建筑物资存量规则

各安全区建筑的物资总量为全地图永久固定，无刷新机制，探索后可查看剩余物资，拿取后建筑内对应物资减少，搜完即空、不可再生，具体存量如下：

- J=居民楼：面包×3、瓶装水×3（仅这两种物资）

- B=办公楼：压缩饼干×4（仅这一种物资）

- S=超市：罐头×5、桶装水×5（仅这两种物资）

- W=水源：清水×6（仅这一种物资）

- M=药店：无任何可拿取物资，仅作为安全区供角色停留、休息

信息可见性与记忆规则：
- 玩家初到一个建筑时，“当前建筑剩余物资”与“建筑内其他角色信息”默认为未知，不能直接获得实时信息。
- 玩家必须执行“探索”后，才会获得该建筑当下“物资快照+角色信息快照”。
- 角色信息快照为单向可见：仅探索者获得该次发现结果，被观察方不会自动收到“自己已被发现”的提示。
- 玩家离开建筑后，上述物资/角色认知会持续保留，不做硬失效删除。
- 当建筑在玩家未参与的情况下发生事件变化（拿取、掉落、进出、战斗导致背包变化）时，玩家旧快照可能失真（STALE），但仍可作为记忆使用。
- 玩家回到建筑时，可直接基于记忆行动（可能失败），也可先探索刷新认知。
- 玩家仅在本人参与相关动作后更新该建筑认知：`EXPLORE`、`TAKE`、`ATTACK`、`GET/TOSS`；`REST`、`USE` 不刷新建筑快照。
- 每轮统一结算后，玩家只更新“本人可见且本人参与可获知”的认知，不获得全局实时信息。

4. 其他基础规则

- 角色状态：水分、饱食（≤0即死）、曝光（仅影响夜晚存活率）
- 角色初始状态与上限（V1）：水分初始 `100`、上限 `100`；饱食初始 `100`、上限 `100`；曝光初始 `0`、上限 `100`。
- 初始背包（V1）：`瓶装水 x1`、`面包 x1`。
- 阶段固定消耗（V1）：每名存活角色每阶段仅扣除一次 `水分-1、饱食-1、曝光±0`。
- 动作额外消耗（V1）：
  - `MOVE`：水分 `-2`、饱食 `-1`、曝光 `+2`
  - `EXPLORE`：水分 `-1`、饱食 `-1`、曝光 `+1`
  - `USE`：水分 `±0`、饱食 `±0`、曝光 `±0`（仅结算物品增益）
  - `TAKE`：水分 `-1`、饱食 `±0`、曝光 `+1`
  - `REST`：水分 `±0`、饱食 `±0`、曝光 `-3`
  - `ATTACK`：水分 `-2`、饱食 `-2`、曝光 `+3`
  - `GET`：水分 `±0`、饱食 `±0`、曝光 `±0`
  - `TOSS`：水分 `±0`、饱食 `±0`、曝光 `±0`
- 夜晚X区存活函数（V1）：`P_survive(E) = clamp(0.97 - 0.008 * E, 0.03, 0.97)`，其中 `E` 为曝光值；每轮在X区抽样 `u∈[0,1)`，`u <= P_survive(E)` 则存活，否则死亡。

- 死亡掉落规则：任意角色死亡时，其背包内剩余物品掉落到“死亡时所在建筑”的库存中；死亡角色背包清空。若该死亡由攻击引发，先执行“胜者夺取”再执行“死亡掉落”。若同轮存在多人死亡，按结算动作先后顺序依次落地到对应建筑库存。

- 回合制：白天、夜晚交替循环；阶段内按轮次推进，每轮由“本阶段仍可行动的存活角色”提交动作并统一结算；若所有存活角色均已阶段结束（死亡或休息），则立即推进下一阶段
- 单人生效规则：当阶段内仅剩1名存活角色时，该角色仍可在同一阶段连续进行多轮动作；仅在其死亡或执行 `REST` 后，该阶段才结束并推进下一阶段。
- 结算顺序规则：同轮动作按“提交时间先后”进入结算，先提交先结算；若提交时间相同，则按固定稳定顺序（如角色ID升序）结算，以保证结果可复现。

- 资源规则：建筑物资固定、搜完即空，不可再生（与上述建筑存量规则呼应）

- 冲突规则：同建筑内可发起攻击，以攻击值判定胜负，为绝境最后手段。攻击值由“基础攻击值+状态修正+信息优势加成”组成。
- 信息优势加成：当 A 已发现 B 且 B 未发现 A 时，A 的攻击值获得 +2；当双方互相发现时，双方均获得 +2（效果等效抵消）。
- 战利品窗口规则：攻击判定后，胜者需在 `GET`（拿取）与 `TOSS`（放弃）中二选一；`GET` 最多从败者背包拿取3件，`TOSS` 则直接放弃本次夺取并继续后续结算。若超时未选，系统默认执行 `TOSS`。
- 攻击值计算（V1）：
  - `AttackScore = Base + StateMod + InfoMod + Rand`
  - `Base = 10`
  - `InfoMod`：信息优势单边生效时 `+2`；互相发现时双方均 `+2`（等效抵消）
  - `Rand`：离散随机项 `{-1,0,+1}`
  - `StateMod = ResourceMod + ExposureMod`
  - `ResourceMod`（令 `R=(水分+饱食)/2`）：`R>=80 => +3`，`60<=R<80 => +2`，`40<=R<60 => 0`，`20<=R<40 => -2`，`R<20 => -4`
  - `ExposureMod`：`曝光<20 => +1`，`20<=曝光<60 => 0`，`60<=曝光<80 => -1`，`曝光>=80 => -2`
- 胜负判定阈值（V1）：令 `Δ = AttackScore(发起方) - AttackScore(被攻击方)`；`Δ>=2` 发起方胜，`Δ<=-2` 被攻击方胜（反杀），`-1<=Δ<=1` 为僵持（无人胜）。
- 胜负强度分级（V1）：`|Δ|=2~3` 为小胜，`|Δ|=4~5` 为中胜，`|Δ|>=6` 为大胜（用于映射后续状态扣减幅度）。

2. 核心循环

房间凑满6人→初始生成角色（坐标、状态、背包）→白天回合→所有角色提交动作（含超时自动处理）→判定是否全员休息→是则直接推进下一轮/否则统一结算→夜晚回合→重复动作提交与判定/结算→循环→每轮结算后按房间 `end_mode` 检查终局条件→满足则立即单局结束并触发终局结算；不满足则继续下一轮

五、玩家操作流程（每轮可执行操作）

玩家每轮仅能选择1个核心动作，严格遵循规则约束，操作步骤：

1. 查看当前信息：坐标、时段、自身状态（水分/饱食/曝光）、背包、当前建筑物资（可能为未知）、同建筑已发现角色（可能为未知）

2. 选择合规动作（6选1）：
        

  - 移动(目标坐标)：仅上下左右相邻1格，且目标坐标不得超出地图边界（x=1~9，y=1~9）；可进入任意地块，进入Q/X后按生死规则结算

  - 探索：仅安全区可执行，探明当前建筑剩余物资与当前建筑角色信息（形成当轮快照并写入记忆）

  - 使用(物品名)：消耗背包物品，补充水分/饱食

  - 休息：降低曝光值（按动作额外消耗表结算），并使本玩家本阶段结束

  - 拿取(物品1,物品2,物品3)：已探索建筑可执行，最多拿3个物资入背包；若记忆与实时库存不一致，可能拿不到或只能部分拿到

  - 攻击(角色名)：仅同建筑且“已被自己发现”的角色可攻击，判定时按“基础攻击值+状态修正+信息优势加成”计算双方攻击值

  - GET(物品1,物品2,物品3) / TOSS：仅在“攻击判定后触发的战利品窗口”由攻击胜者执行，不计入本轮“6选1”核心动作

3. 提交动作，等待所有角色（真人+AI）动作提交完毕，进入下一步判定/结算

4. 超时处理：若玩家在规定时间内未提交任何动作，系统自动为该玩家执行“休息”动作；AI Agent无超时情况，系统按其决策逻辑自动提交动作，无需玩家干预。

六、AI Agent决策逻辑（系统自动执行）

AI以活下去为唯一目标，系统自主决策，无玩家干预，且无操作超时情况：

- 优先保命：绝不进入白天Q/X，不做让水分/饱食归零的操作

- 时段策略：白天主探索/移动/收集物资，夜晚主休息/隐蔽，少移动无攻击

- 资源优先：水分/饱食不足时，优先探索、拿取、使用物资

- 冲突规避：仅濒临死亡、有绝对状态优势时才攻击，否则优先撤离

- 消耗计算：所有动作先核算消耗，确保行动后存活

七、结算系统核心功能（核心作用）

结算系统为游戏核心中枢，负责状态计算、生死判定、资源更新、终局结算，全流程无人工干预，核心触发场景分为两种：

1. 常规结算（非全员休息场景）

每轮由本阶段仍可行动的存活角色提交动作（含超时自动休息）后，若未出现“所有存活成员均阶段结束”的情况，系统自动执行常规结算：

- 基础消耗结算：扣除每阶段全局固定消耗（每名存活角色在该阶段仅扣1次），并叠加本轮动作额外消耗（按动作额外消耗表覆盖 `MOVE/EXPLORE/USE/TAKE/REST/ATTACK/GET/TOSS`）。

- 状态与物资结算：计算“使用物品”后的水分/饱食增加值；更新“拿取/探索”对应的建筑剩余物资、角色背包物资；结算“休息”的曝光扣减效果；判定“攻击”成败并执行胜负效果（状态扣减、曝光增减、战利品窗口）。
- 共享资源冲突结算：同轮多人对同一建筑库存执行拿取时，按结算顺序逐个扣减库存；后结算角色可能出现“拿不到或部分拿到”。
- 攻击战利品结算：攻击判定后，胜者（可为发起方或反杀方）必须在战利品窗口执行 `GET` 或 `TOSS`；`GET` 最多可从败者背包取走3件，`TOSS` 为放弃夺取。战利品窗口结束后，再进入生死判定与死亡掉落流程；若超时未选，系统默认 `TOSS`。

- 生死判定结算：按生死铁律统一判定——白天处于Q/X→立即死亡；水分≤0或饱食≤0→立即死亡；夜晚处于X→按 `P_survive(E) = clamp(0.97 - 0.008 * E, 0.03, 0.97)` 与随机抽样结果判定存活/死亡；移除死亡角色，标记存活角色。

- 死亡背包落地结算：角色被判定死亡后，立即执行其背包物资落地到当前建筑库存，并将该角色背包清空；该变化参与后续玩家可见库存与探索结果。

- 回合推进：结算完成后，切换时段（白天↔夜晚），开启下一轮。

2. 特殊推进（全员休息场景）

若本轮所有角色（真人+AI）均执行“休息”动作（含真人超时自动休息、AI自主选择休息），则触发特殊推进规则：

- 若该阶段基础消耗尚未扣除，则仅执行一次每阶段全局固定消耗（所有存活角色水分-1、饱食-1）；无需执行其他额外结算（无动作额外消耗、无状态/物资额外变动）。

- 直接推进至下一轮，同步切换时段（白天↔夜晚），无需等待完整结算流程，节省回合耗时。

3. 终局结算

- 触发条件（按房间 `end_mode` 判定）：
  - `ALL_DEAD`：任一结算节点判定房间内“总存活数=0”时，立即触发终局结算。
  - `HUMAN_ALL_DEAD`：任一结算节点判定房间内“真人存活数=0”时，立即触发终局结算。

- 结算内容：1. 统计所有角色：存活轮次、获取物资总量、死亡原因；2. 统计真人玩家战绩（最后存活真人、真人存活时长、真人击杀/被击杀）；3. 生成房间排名（存活轮次从高到低）；4. 重置房间所有数据（角色、物资、坐标），支持开启下一局。

八、单局结束条件

- `ALL_DEAD`：房间内所有角色（真人+AI）均死亡时，单局结束。
- `HUMAN_ALL_DEAD`：房间内所有真人玩家均死亡时，单局结束（即使仍有AI存活也结束）。
