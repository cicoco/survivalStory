## 2. 规划边界

- 本文档仅定义 V2+ 的演进方向。
- 不改变 V1 已确认实现范围与验收口径。
- MAPPO 明确为 V2+ 规划能力，不纳入 V1 实现范围。

## 3. AI 策略演进

- 统一策略接口：`choose_action(obs, action_mask) -> action`
- 分阶段策略路线：`RuleBot -> LLMPolicy -> MAPPO`
- LLMPolicy 采用两层机制（发现层 + 意图层），并保留 RuleBot 兜底。
- MAPPO 采用离线训练 + 在线推理，按灰度策略逐步接入。

## 4. 观测特征演进

- 引入 `local_map_memory_3x3` 与 `last_seen_round_norm`（V2+）。
- 支持可变地图尺寸与布局配置，避免固定 9x9 耦合。
- 可将物品效果参数（如 `food_gain/water_gain`）显式输入策略，以降低平衡改动时重训成本。

## 5. 数据与运维演进

- 在保留 V1 内存态方案基础上，按需引入 PostgreSQL：
  - 战绩持久化
  - 审计日志
  - 回放索引
- 增加进程级快照落盘与恢复能力，降低重启丢局风险。

## 6. 体验与分析演进

- 可选提供回放导出能力（文件下载与复盘回看）。
- 建立 AI 质量指标看板：
  - Discovery 成功率
  - Intent 成功率
  - 回退率
  - 非法动作率
  - 推理延迟分位数

## 7. 里程碑建议

1. V1.1：LLMPolicy 灰度 + 指标看板
2. V1.2：特征增强（局部记忆层）+ 可变地图准备
3. V2.0：MAPPO 灰度上线 + PostgreSQL 持久化
